Lesson 1: The Basics of the Web and HTML

The World Wide Web is a collection of HTML documents. The browser on your computer makes requests to servers. Servers are computers that host the files that make up the web. They communicate using HTTP, the main protocol of the web.

HTML stands for Hyper Text Markup Language. HTML consists of the text or content of the page, the markup or what it looks like/how it's arranged, references to other documents such as images or videos, and lastly links to other pages on the internet. For example, here is a link to the Udacity Course Catalog.

In Lesson 1 we learned a lot about different tags and their attributes. Tags are usually made up of opening and closing tags. Void tags, however, do not need a closing tag as they do not contain content. Examples of void tags include the < br > tag. It's a good idea to make your closing tags as soon as you have typed your opening tags. If you forget, the computer won't know to fix your mistake and the webpage will not display correctly.

We also learned about the difference between inline and block elements. Inline elements are simply text elements, such as the img or br tags whereas block elements, such as the paragraph tag, make an invisible block around the content.

The span and div tags are important because we can attach styles to them. The span tag is an inline element and the div tag is a block element.

The 5 Biggest Takeaways From This Lesson

1. A webpage is a text document written in HTML. Web browsers read these documents, interpret, and display them.

2. Coding is the language the computer understands

3. Computers are stupid. Programmers need to be exact with their language. A simple typo could prevent the webpage from displaying correctly or at all.

4. Programmers can not remember everything. It is okay to look up bits of code.

5. Basic HTML vocabulary, including tags and elements.
